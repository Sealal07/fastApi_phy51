# Разбор проекта Space Control


## 1. Слой базы данных (`database.py`)

В этом модуле настраивается взаимодействие с PostgreSQL через ORM SQLAlchemy.

* **Модели данных**: Мы описываем таблицы как классы Python. `UserModel` отвечает за учетные записи, а `SpaceshipModel` — за характеристики кораблей.
* **Безопасность типов**: Используется современный синтаксис `Mapped` и `mapped_column`, который позволяет Python понимать типы данных в базе (например, `float` для топлива или `str` для названия).
* **Управление сессиями**: Функция `get_db` — это генератор, который создает сессию для каждого запроса и закрывает её после завершения, предотвращая утечку ресурсов.

---

## 2. Безопасность и JWT (`auth.py`)

Этот файл управляет "пропусками" в нашу систему.

* **Хеширование паролей**: Используется библиотека `passlib` с алгоритмом `bcrypt`. Пароли никогда не хранятся в открытом виде; вместо "12345" в базу попадает длинный зашифрованный след.
* **JWT (JSON Web Token)**: Когда пользователь входит в систему, функция `create_access_token` генерирует зашифрованную строку (токен). В этот токен "зашито" имя пользователя и время истечения (30 минут).
* **Защита маршрутов**: Функция `get_current_user` проверяет токен, присланный клиентом. Если токен неверный или срок его действия истек, сервер выдает ошибку `401 Unauthorized`.

---

## 3. Логика API (`space_api.py`)

Здесь описано, как сервер обрабатывает входящие запросы.

* **Валидация через Pydantic**: Класс `Spaceship` гарантирует, что мы не получим корабль с отрицательной скоростью (`gt=0`) или некорректным уровнем топлива (`ge=0, le=100`).
* **Внедрение зависимостей (DI)**: С помощью `Depends()` функции получают доступ к базе данных или проверяют авторизацию текущего пользователя "на лету".
* **Бизнес-логика**: При регистрации корабля сервер сам вычисляет `estimated_arrival_hours`, деля расстояние до Марса на скорость корабля. Это избавляет фронтенд от лишних расчетов.

---

## 4. Клиентская часть (`App.jsx`)

React-приложение взаимодействует с нашим API через библиотеку `axios`.

* **Состояние (State)**: Используются хуки `useState` для хранения списка кораблей, данных форм и токена авторизации.
* **Жизненный цикл**: Хук `useEffect` вызывает функцию `fetchShips` сразу после открытия страницы, чтобы пользователь увидел актуальный список флота.
* **Авторизация на клиенте**: Токен сохраняется в `localStorage`. При отправке данных о новом корабле, токен извлекается и добавляется в заголовок `Authorization: Bearer <token>`.

---

## Как запустить и изучить
1.  **Проверьте БД**: Убедитесь, что в `database.py` указаны верные данные от вашего PostgreSQL.
2.  **Запустите сервер**: `uvicorn space_api:app --reload`.
3.  **Изучите Swagger**: Перейдите на `http://127.0.0.1:8000/docs`. Там можно протестировать каждый эндпоинт без фронтенда.
4.  **Запустите фронт**: В папке `space-front` выполните `npm start` или `npm run dev`.

## Подробный разбор функций

### 1. Файл `auth.py`  

* **`verify_password(plain, hashed)`**: Сравнивает пароль, введенный пользователем, с хешем из базы данных. Использует безопасный алгоритм, чтобы предотвратить атаку по времени.
* **`get_password_hash(password)`**: Превращает "сырой" пароль в хеш. Вызывается один раз при регистрации пользователя.
* **`create_access_token(data)`**: Формирует JWT-токен. Она берет `sub` (имя пользователя), добавляет время жизни (`exp`) и подписывает это секретным ключом `SECRET_KEY`.
* **`get_current_user(token)`**: Главная функция-фильтр. Она извлекает токен из заголовка, расшифровывает его и проверяет, существует ли такой пользователь. Если токен подделан, она "выбрасывает" исключение 401.

### 2. Файл `database.py`  

* **`create_db()`**: Создает все таблицы в PostgreSQL на основе описанных классов (`UserModel`, `SpaceshipModel`). Вызывается один раз при старте приложения.
* **`get_db()`**: Функция-генератор для управления сессиями. Она открывает соединение с БД для каждого запроса и закрывает его сразу после выполнения кода, чтобы не перегружать базу.

### 3. Файл `space_api.py`  

* **`register_user(user, db)`**: Принимает JSON с данными нового пользователя, проверяет через запрос `db.query`, не занято ли имя, хеширует пароль и сохраняет запись в таблицу `users`.
* **`login(form_data, db)`**: Обрабатывает вход. Она ищет пользователя в базе, проверяет пароль через `verify_password` и, если всё верно, выдает JWT-токен.
* **`get_all_ships(db)`**: Выполняет простой запрос `query(SpaceshipModel).all()` и возвращает список всех кораблей. Доступна всем пользователям без авторизации.
* **`register_ship(ship, db, current_user)`**: Самая сложная функция. 
    1. Требует токен (зависимость `get_current_user`).
    2. Рассчитывает время полета по формуле `distance / speed`.
    3. Сохраняет корабль в базу, привязывая его (логически) к имени пользователя, который сделал запрос.

### 4. Файл `App.jsx` 

* **`fetchShips()`**: Выполняет GET-запрос к API и обновляет состояние `ships`, что заставляет React автоматически перерисовать список на экране.
* **`handleRegister(e)`**: Отправляет данные из формы регистрации на сервер. Если всё успешно, выводит уведомление.
* **`handleLogin(e)`**: Отправляет логин и пароль в формате `FormData` (это стандарт для OAuth2 в FastAPI). В случае успеха сохраняет полученный токен в `localStorage` и в переменную состояния `token`.
* **`handleShipSubmit(e)`**: Собирает данные о новом корабле и отправляет POST-запрос. Важный момент: она добавляет токен в заголовок `Authorization`, иначе сервер отклонит запрос.
* **`logout()`**: Просто очищает токен из памяти браузера и состояния приложения, тем самым скрывая защищенные формы.
